# note-md 本番運用向け ペースト手順

最終更新日: 2026-03-14

この文書は VS Code 拡張機能 note-md を使って、Markdown 原稿を note.com に移すときの実運用フローを整理したものです。

## 目的

- note で崩れにくい書式だけを使う
- ローカル画像は必要に応じて自動変換・自動アップロードする
- 本文コピー時の貼り付け事故を減らす

## 推奨フロー

1. Markdown 原稿を開き、プレビューで見た目と警告を確認する。
2. ローカル画像がある場合は、プレビューを開いた状態で自動処理を待つか、必要なら強制再アップロードする。
3. 文字数カウンターとプレビュー表示を確認する。
4. 「タイトル」ボタンでタイトルをコピーし、note のタイトル欄へ貼る。
5. 「本文コピー」ボタンで本文をコピーし、note エディタへ貼る。
6. note 側で最終確認する。

## 貼り付け前の確認項目

- h1 はタイトルとして分離されているか
- h2 / h3 の見出し構造が意図通りか
- 箇条書き・番号付きリストが崩れていないか
- コードブロックの言語指定と改行が壊れていないか
- Mermaid / 数式 / ルビが想定どおりか
- 画像 URL の置換が完了しているか

## 画像の扱い

- ローカル画像がなければ、本文コピーはすぐに有効になる
- note 非対応形式の画像は PNG に変換してからアップロードする
- アップロード結果は VS Code セッション内キャッシュで再利用する
- 同じ画像内容を別パスから参照していても、本文コピーでは各参照先が正しく置換される

## 失敗時の戻し方

- プレビュー上で画像が未反映なら、再度保存して自動処理を待つ
- サービス障害が疑わしい場合は「強制再アップロード」を使う
- note 側で崩れた場合は、まず本文だけを貼り直し、画像は個別に挿入する

## この文書が扱わないもの

- 旧 CLI ベースの RTFD / HTML 変換運用
- TextEdit や外部スクリプトを使った貼り付け手順

それらは現行拡張のリリース対象ではなく、運用の正本はこの文書と README に統一する。

見出し画像は本文画像とは別管理です。
本文の移行作業と分けて最後に設定した方が混乱しません。

## 投稿後の確認チェックリスト

投稿後、少なくとも次を確認します。

1. 画像の挿入順が本文の説明順と一致しているか
2. 図の文字がスマートフォン幅でも読めるか
3. 見出し画像が別途設定されているか
4. 画像をファイル添付にしてしまっていないか

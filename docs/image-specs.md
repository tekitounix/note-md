# note の画像仕様メモ

最終確認日: 2026-03-14

この文書は、note に画像を載せるときの仕様を、公式ヘルプをもとに集約したメモです。
特に、本文中の図版、見出し画像、クリエイターページのヘッダー画像、プロフィール画像、メンバーシップ画像まわりを対象にしています。

Markdown 原稿を note に移す実務手順は、[paste-workflow.md](paste-workflow.md) にまとめています。

## 要点

| 項目 | 現行の把握 |
|---|---|
| 本文に挿入できる画像形式 | JPG、PNG、GIF、HEIC |
| 本文画像の容量上限 | 20MB |
| 本文画像の表示幅 | 横幅 620px を超えると自動で 620px に縮小 |
| 拡大表示 | 長辺 4000px まで対応 |
| SVG | 本文画像形式に含まれていないため非対応扱い |
| 登録画像の共通挙動 | アップロード時に最適化（圧縮）される |
| EXIF | 自動削除 |
| 画質対策 | note 公式ヘルプでは PNG を試すことを案内 |

## 1. 本文画像

### 形式

本文に挿入可能な画像形式は JPG、PNG、GIF、HEIC です。
SVG はヘルプに含まれていないので、非対応と考えるのが安全です。

### 容量と表示幅

本文に挿入する画像の最大サイズは 20MB です。
また、横幅 620px を超える画像をアップロードすると、自動で横幅 620px に縮小されます。

つまり、本文に表示される実幅は 620px 基準です。

> **注**: 以下は公式ヘルプには明示されていない推論です。
> 拡大時に 4000px まで対応していることから、アップロード画像は原寸で保存されていると推測できます。
> Retina など高密度表示では 620px ちょうどの画像は粗く見えやすいので、見た目の品質を重視するなら 2 倍以上のピクセル数で用意し、note 側に縮小させる方が有利です。

### 拡大表示

画像拡大時は、長辺 4000px まで対応しています。
そのため、本文表示は 620px でも、拡大表示時の見え方まで考えるなら、ある程度大きい画像を持っておく意味があります。

### 小さく表示する機能

本文に挿入した画像は、エディタ上で 60% に縮小表示できます。
PC では画像をクリックするとツールバーが出て、その中の縮小ボタンで切り替えます。
モバイルでも操作できますが、投稿完了後の見た目確認は PC で行う案内になっています。

### 表示上の補足

白背景と画像の境界が分かりにくい場合、本文中の画像にはグレーの枠線が表示されます。

## 2. 登録画像の共通仕様

note にアップロードした画像は、登録時に最適化されます。
ヘルプでは「圧縮」と説明されています。
画像によっては、色味や画質に影響が出る場合があります。

また、EXIF 情報は削除されます。
位置情報や撮影日時などを残したい用途には向きません。

プロフィール画像やヘッダー画像で画質が落ちる場合について、note ヘルプは「システムで一度画像を処理する関係で、画質が若干落ちることがある」と説明しています。
その対策として、PNG 形式を試すことが案内されています。

## 3. 記事の見出し画像

現行ヘルプの「登録画像の推奨サイズ一覧」では、記事の見出し画像の推奨サイズは 1280 x 670 px です。
画像記事の 1 枚目も、タイムライン上ではこの規格でトリミングされると説明されています。

一方で、note 公式の古い記事には、テキストの表紙画像は 1280 x 530 px で表示され、白銀比に基づいて中央トリミングすると書かれています。
この情報は現在のヘルプと食い違っています。

そのため、現時点では次の整理が安全です。

| 観点 | 採用すべき目安 |
|---|---|
| 現行の運用に合わせる | 1280 x 670 px |
| 古い仕様の参照 | 1280 x 530 px の中央トリミング説明が存在 |

実務上は、現行ヘルプを優先して 1280 x 670 px を使うのが無難です。
ただし、中央トリミング前提の考え方自体は今も有効なので、重要な要素は中央寄せにしておく方が安全です。

## 4. クリエイターページ / マガジンのヘッダー画像

ヘッダー画像の推奨比率は 1.91:1 です。

基本サイズは 1280 x 670 px です。
よりきれいに表示したい場合は 1920 x 1006 px が推奨されています。

ヘルプでは、次の事情が明記されています。

| 項目 | 内容 |
|---|---|
| PC と mobile | 表示サイズが変わる |
| クリエイターページ / マガジン | 中央部分が切り取られて表示される |
| マガジントップ / X シェア | 画像全体が表示される |
| マガジン一覧ページ | 1280 x 454 px の範囲が表示される |

つまりヘッダー画像は、単に 1.91:1 の画像を作るだけでは足りず、中央安全領域を意識する必要があります。

## 5. プロフィールアイコン

プロフィールアイコンには明確な推奨サイズはありません。
ただし、正方形画像が推奨です。

表示時は丸い枠に合わせてトリミングされます。
位置調整と拡大縮小ができるので、中央に主要な要素を置く前提で画像を用意するのが安全です。

## 6. メンバーシップ画像と会員証

メンバーシップ画像は 1280 x 720 px が推奨です。
よりきれいに表示したいなら 1920 x 1080 px が推奨されています。

会員証の画像は 1280 x 720 px です。

## 7. スキのリアクション画像

スキのリアクションには JPG、PNG、GIF を使えます。
アニメーション GIF も利用可能です。

推奨サイズはありませんが、容量は 10MB までです。
長い GIF は吹き出しが消える前に再生し切れない場合があります。

## 8. 音声記事のサムネイル

音声記事のサムネイル画像は正方形で表示されます。
10MB 以内で、1000 x 1000 px 以上が推奨されています。

## 9. 画像記事と GIF の扱い

本文中の画像挿入では GIF が使えます。
ただし画像記事は GIF 非対応です。
GIF を使いたい場合は、テキスト記事の本文に挿入する必要があります。

## 10. ファイルアップロードとの違い

記事中に画像を見せたい場合は、ファイル添付ではなく本文画像として挿入する必要があります。

ファイルアップロード機能は、記事中に添付ファイルを置くためのもので、1 ファイル 50MB までです。
画像ファイルをファイル添付として入れた場合、本文中に直接表示されず、ダウンロード形式になります。

つまり、図版を本文中で見せたい用途では、ファイルアップロード機能は代替になりません。

## 11. 実務向けの判断基準

### 本文中の図版

本文の実表示幅は 620px ですが、620px ぴったりの PNG は高密度画面で粗く見えやすいです。
図中に文字や細線がある場合は、1240px 以上で書き出して note 側に縮小させる方が見栄えが安定します。

特に横長で高さの低い図は、文字がつぶれやすいので、1240px でも足りなければ 1860px や 2480px を検討した方がよいです。

### ヘッダー・プロフィール系

アップロード時に圧縮がかかるので、画質が重要なものは PNG を優先した方が安全です。
また、トリミング前提の表示が多いため、重要な要素は中央に置くべきです。

## 12. このリポジトリでの運用メモ

このリポジトリでは、SVG を元に図版を作り、note 投稿用には PNG に変換するのが扱いやすいです。
理由は次のとおりです。

| 項目 | 理由 |
|---|---|
| SVG | 元データとして編集しやすい |
| PNG | note の本文画像形式として安全 |
| GIF | 静止図では不要 |
| HEIC | 環境差が出やすく、図版用途に向かない |
